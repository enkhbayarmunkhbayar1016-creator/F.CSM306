# Lab01 – Parallel Sum (F.CSM306)

## 1. Лабораторийн зорилго
Энэ лабораторийн ажлын зорилго нь олон цөмт CPU ашиглан тооцооллыг хэрхэн хурдсгаж болохыг судлах юм.  
Нэг ижил бодлогыг дараалсан (sequential) болон параллел аргуудаар бодож, гүйцэтгэлийн хугацаа, хурдны өсөлт (speedup), үр ашгийг (efficiency) харьцуулж судална.

## 2. Бодлогын тайлбар
80,000,000 элементтэй `double` төрлийн массив үүсгэнэ.  
Массивын бүх элемент нь 1.0 утгатай байна.  
Энэ массивын бүх элементийн нийлбэрийг дараах 3 аргаар тооцоолно.

1. Sequential (нэг цөм ашиглах)
2. std::thread (олон thread ашиглах)
3. OpenMP (parallel for + reduction)

Эдгээр аргуудын гүйцэтгэлийг хооронд нь харьцуулна.

## 5. Ашигласан аргуудын тайлбар

### 5.1 Sequential
Нэг CPU цөм ашиглан массивын бүх элементийг дарааллаар нь нийлбэрлэнэ.  
Энэ арга нь хэрэгжүүлэхэд хамгийн энгийн боловч гүйцэтгэл хамгийн удаан.

### 5.2 std::thread
Массивыг хэд хэдэн хэсэгт хувааж, хэсэг бүрийг тусдаа thread дээр бодно.  
Thread бүр өөрийн хэсгийн нийлбэрийг бодоод, эцэст нь бүгдийг нэгтгэнэ.  
Sequential аргыг бодвол илүү хурдан гүйцэтгэнэ.

### 5.3 OpenMP
OpenMP-ийн `parallel for reduction` ашиглан нийлбэрийг параллелаар бодно.  
Thread үүсгэх болон синхрончлолыг компилятор автоматаар зохицуулдаг.  
Энэ лабораторид хамгийн сайн гүйцэтгэл үзүүлсэн.

## 6. Хугацаа хэмжилт
Хугацааг `std::chrono::steady_clock` ашиглан секундээр хэмжсэн.  
Бодит хэмжилтээс өмнө warm-up хийж CPU-г бэлтгэсэн.  
Хэмжилтийн үр дүнг `results.csv` файлд хадгалсан.
